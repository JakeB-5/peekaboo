## Peekaboo とは？

Peekaboo は任意のウェブページを、半透明でサイズ変更可能なウィンドウとして、今やっている作業の上に浮かべます。デフォルトの **ゴースト（ghost）** 状態ではほぼ見えず、クリックはそのまま下のアプリへ通過します。カーソルをホットゾーンに入れるとコンテンツがくっきりし、**グローバルホットキーを押すと痕跡なく即座に消えます。**

製品の本質は一行に集約されます ——「**見たい時だけ見え、必要な瞬間に即座に消える。**」

## 機能

| | 機能 | 内容 |
|:--:|---|---|
| 👻 | **ゴーストオーバーレイ** | あらゆるものの上に浮かぶ半透明・枠なし・常に最前面のウィンドウ |
| 🫥 | **ホバー表示** | 普段は低不透明度＋クリックスルー、カーソルがホットゾーンに入るとくっきり |
| 🚨 | **パニックホットキー** | システム全体のショートカット一発で即座に非表示 — 非フォーカスでも動作 |
| 🙈 | **アプリの隠蔽** | Dock・Cmd-Tab・メニューバーから非表示（Accessory アプリ） |
| 🎚️ | **不透明度コントロール** | 「通常」と「ホバー」の不透明度を個別に設定 |
| 🖥️ | **画面共有での非表示** | best-effort のみ — 下記の正直な注意点を参照 |

## 使い方

> 以下の画像は Peekaboo の実際のインターフェース（アプリ自身の HTML/CSS）から生成した **UI レンダー** です。実際のスクリーンショットに差し替えるには macOS で撮影してください —— content protection は macOS 14 以下でウィンドウをスクリーンショットから除外することがあるため、設定で **画面共有での非表示** を先にオフにしてください。

**1 · ページを浮かべる。** Peekaboo を起動すると、薄い常に最前面の **ゴースト** オーバーレイとして開きます。通常の不透明度ではほとんど見えず、クリックはそのまま下のアプリへ通過します。

![デスクトップの上に薄く浮かぶゴースト状態の Peekaboo](docs/images/usage-ghost.png)

**2 · ホバーで表示。** カーソルをオーバーレイのホットゾーンに入れると、コンテンツがくっきり（フル不透明）と表示されます。離れると再びゴーストに戻ります。

![カーソルがホットゾーンに入りフル不透明で表示された Peekaboo](docs/images/usage-reveal.png)

**3 · 調整する。** ☰ ボタンで設定を開きます —— URL・ブックマーク、通常／ホバーの不透明度、ウィンドウサイズ、3×3 ホットゾーン、パニックホットキー、隠蔽トグルを設定できます。

![Peekaboo の設定パネル](docs/images/usage-settings.png)

**4 · 消す。** パニックホットキー（既定 ⌘⇧H）または ✕ ボタンを押すと、別のアプリにフォーカスがある状態でもウィンドウが即座に消えます。もう一度押すと、見ていた位置のまま復帰します。

![パニックホットキーで Peekaboo が隠れた後のデスクトップ](docs/images/usage-panic.png)

## 仕組み

責任が明確に分かれた Tauri v2 アプリです。**ステルスの本体はすべて Rust コアが担当** します —— ウィンドウの透明化・常に最前面・クリックスルー・グローバルパニックホットキー・ホバー検出のためのカーソルポーリング・表示/パニックのステートマシン。**WebView は描画のみ**（ウェブコンテンツのレンダリングと設定 UI）を担います。

難しいのは UI ではなく macOS のウィンドウ制御です。オーバーレイがクリックスルー状態だと通常の DOM・ネイティブのホバーイベントが発火しないため、ホバー表示はグローバルなカーソル座標をホットゾーンの矩形と比較するポーリングで実現します。

## ⚠️ 正直な注意点: 画面共有

macOS の content protection による画面共有での非表示は、**macOS 15（Sequoia）以降では機能しません。** Zoom・Meet・QuickTime が使う ScreenCaptureKit が合成済みフレームバッファを取得し、`NSWindowSharingNone` フラグを無視するためです（[Tauri #14200](https://github.com/tauri-apps/tauri/issues/14200)、既知の回避策なし）。旧 macOS（≤ 14）と従来のスクリーンショット API に対してのみ有効です。

したがって画面共有での非表示は **補助手段（best-effort）** に留め、**画面共有時の実質的な防御線はパニックホットキー**（共有の前に、または即座に隠す）とします。
