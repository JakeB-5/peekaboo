Peekaboo — 개요 · 인덱스

Peekaboo는 눈에 띄지 않게 웹을 보기 위한 macOS 데스크톱 스텔스 브라우저 오버레이 앱이다. 제품의 본질은 한 줄로 압축된다 — "보고 싶을 때만 보이고, 위험할 때 즉시 사라진다." 임의의 웹 페이지를 원하는 크기·투명도의 떠 있는 창으로 띄우되, 평소엔 거의 드러나지 않다가 지정 영역에 마우스를 올릴 때만 또렷해지고(hover-reveal), 들킬 위기엔 전역 단축키 한 번으로 사라진다 (화면 공유 비노출은 best-effort — 최신 macOS에선 한계). 본 인덱스는 5개 상세 문서의 허브로서 비전·핵심 결정·지표·진입점을 한 화면에 모은다.

플랫폼 = Tauri v2 타깃 = macOS 단독 핵심가치 = 안 들킴 현재 = greenfield(코드 0)

① 한 줄 비전 · 배경

비전: "누구에게도 들키지 않고, 보고 싶을 때 즉시 띄워 잠깐 보고, 인기척이 느껴지면 단축키 한 번에 흔적 없이 사라지는 웹 뷰어." 일반 브라우저로 보는 것과의 차이는 전부 은닉성에 있다 — 평소 노출 최소화, 위기 시 즉시 탈출, 화면 공유 비노출(best-effort), 앱 존재 자체의 은폐.

배경: 노출된 환경에는 "들킬 경로"가 여럿이다. ⑴ 화면 공유·녹화 중 화면에 떠 있던 창이 그대로 송출되고, ⑵ 누군가 등 뒤로 다가오며, ⑶ 평소 화면에 콘텐츠가 빤히 떠 있고, ⑷ Dock·Cmd-Tab·앱 스위처에 앱 흔적이 남는다. Peekaboo는 이 네 경로 각각에 대응하는 방어 기제를 OS 레벨(Tauri/Rust)에서 구현하는 것을 목표로 한다. 난이도는 UI가 아니라 macOS 창 동작 제어에 있다.

무엇을

투명도·크기 조절되는 항상-위 웹 오버레이 + hover-reveal + 패닉 단축키 + 화면공유 비노출(best-effort).

왜

일반 브라우저로는 불가능한 "안 들키는" 사용 — 노출 최소화·즉시 탈출·흔적 은폐.

어떻게

Tauri v2(Rust 코어 + WebView UI)로 macOS 창 속성을 직접 제어. 경량 바이너리 = 흔적 최소.

② 위협 모델 · 방어 기제

"들킬 수 있는 경로"를 네 가지로 정의하고, 각 경로에 1:1로 대응하는 방어 기제를 둔다. 이 매핑이 제품 전체의 설계 골격이며, 각 방어 기제의 구현은 아키텍처·동작 문서에서 OS API 레벨로 상세화한다.

현실 점검 · 화면 공유 비노출의 한계

사전 검증 결과, content protection은 macOS 15(Sequoia) 이상에서 화면 공유·녹화를 막지 못한다 — Zoom·Meet·QuickTime가 사용하는 ScreenCaptureKit가 합성된 프레임버퍼를 그대로 캡처하며 NSWindowSharingNone 플래그를 무시하기 때문이다(Tauri #14200, Apple 측 의도된 변경, 알려진 우회 없음). content protection은 구버전 macOS·구형 캡처 API에서만 유효하다. 따라서 화면 공유 상황의 실질적 방어선은 "패닉 단축키로 사전/즉시 숨기기"이며, content protection은 보조 수단으로만 둔다. 근거와 대응은 아키텍처·로드맵(리스크)에서 다룬다.

③ 핵심 결정

권고 · Tauri v2 + macOS 단독

1. 프레임워크는 Tauri v2(Rust 코어 + 시스템 WebView). 투명·항상-위·글로벌 단축키·클릭 통과·화면공유 비노출이 모두 네이티브 창 속성으로 제어 가능하고, 바이너리/메모리가 가벼워 "흔적 최소화"라는 제품 가치와 일치한다.

2. 타깃은 macOS 단독. 투명창·content protection·Dock 은폐 등 핵심 기능이 OS별로 동작이 갈리므로, 크로스플랫폼 추상화 비용을 들이지 않고 macOS 전용 API에 의존해 완성도를 높인다.

3. 책임 경계는 명확히 둔다 — 스텔스의 본체는 Rust(Tauri) 측 창 제어이고, WebView(프론트)는 표현·상호작용만 담당한다. 이 경계가 아키텍처 설계의 중심이다.

Phase 0–1 · 스캐폴딩 + MVP

검증 원칙

스텔스 앱의 합격선은 "기능이 된다"가 아니라 "들키지 않는다"이다. 각 방어 기제는 실제 위협 시나리오로 검증한다 — 화면 공유 시 창이 보이지 않는가 / 다른 앱이 포커스인 상태에서 패닉 키가 즉시 듣는가 / Dock·Cmd-Tab에 안 뜨는가 / hover 밖 클릭이 아래로 통과하는가. macOS 권한(필요 시)·동작은 버전에 따라 다를 수 있어 아키텍처 문서에서 실제 API로 확인한다.
