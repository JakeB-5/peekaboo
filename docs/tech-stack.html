기술스택 · 구조 — Peekaboo

구현에 바로 쓰는 스택·디렉터리·설정·권한·코드 골격. 모든 식별자(설정 키, Rust 메서드, 권한 ID, CLI)는 사전 검증값이다. 동작 설계의 근거는 아키텍처, 기능 우선순위는 기능 명세, 실행 순서는 로드맵 참조.

Tauri v2 · Rust Frontend = TS + Vite plugin = global-shortcut feature = macos-private-api

①스택 개요

최소 의존성 원칙. 스텔스 기능은 Tauri 코어 + 공식 global-shortcut 플러그인만으로 충족되며, 프론트는 단순 표현이라 무거운 프레임워크가 필요 없다.

Tauri v2 코어

필수
언어 RustWebView 시스템(WKWebView)

창 속성 제어·IPC·번들의 본체. 투명 창을 위해 macos-private-api feature를 켠다.

tauri = { version = "2", features = ["macos-private-api"] }

global-shortcut 플러그인

필수
Rust tauri-plugin-global-shortcutJS @tauri-apps/plugin-global-shortcut

패닉 단축키. 표준 조합은 권한 없이 비포커스에서도 발화.

추가: cargo tauri add global-shortcut

Frontend (TS + Vite)

표현
역할 콘텐츠·투명도·설정 UI

웹 콘텐츠 렌더와 노출 애니메이션, 설정 패널. 상태는 Rust가 보유하므로 프론트는 이벤트 수신·표현만. vanilla 또는 경량 프레임워크.

선택: tauri-plugin-store(설정 영속화)

사전 준비

Rust 툴체인(rustup), Node.js(LTS), macOS의 Xcode Command Line Tools(xcode-select --install)가 필요하다. 스캐폴딩은 npm create tauri-app@latest.

②프로젝트 구조

Tauri v2 표준 레이아웃. 프론트(src/)와 Rust 코어(src-tauri/)가 분리되고, 권한은 capabilities/에 선언한다.

peekaboo/
├─ src/                      # 프론트엔드 (TS + Vite) — 표현 담당
│  ├─ index.html
│  ├─ main.ts                # 이벤트 수신 · 설정 UI · 명령 invoke
│  └─ styles.css             # 투명도/노출 상태 CSS
├─ src-tauri/
│  ├─ Cargo.toml             # tauri(features: macos-private-api) + 플러그인
│  ├─ tauri.conf.json        # 창 · 은폐 · 번들 설정
│  ├─ build.rs
│  ├─ capabilities/
│  │  └─ default.json        # 권한 부여(window·global-shortcut)
│  ├─ icons/
│  └─ src/
│     ├─ main.rs             # 엔트리 → lib::run()
│     └─ lib.rs              # 플러그인 · 패닉 단축키 · 커서 폴링 · 상태머신
├─ package.json
└─ vite.config.ts
스텔스 관점의 구조 주의

productName·번들 식별자·앱 이름·설정 파일 경로가 "흔적"이 된다. 배포물 이름은 눈에 띄지 않게 평범히 두고, 영속화 파일도 일반적인 위치/이름을 쓴다(예: 앱 식별자를 노골적으로 "Peekaboo"로 노출하지 않는 선택지 고려).

③tauri.conf.json 핵심

오버레이·은폐의 거의 전부가 창/앱 설정에서 출발한다. 아래는 핵심 키만 추린 예시다.

{
  "productName": "Peekaboo",
  "identifier": "com.jakeb5.peekaboo",
  "app": {
    "macOSPrivateApi": true,                  // 투명 창에 필수(+ Cargo feature)
    "macOS": { "activationPolicy": "accessory" }, // Dock/Cmd-Tab 숨김(런타임 대안도 있음)
    "windows": [
      {
        "label": "main",
        "url": "https://example.com",
        "width": 420, "height": 720,
        "transparent": true,                   // 투명 배경
        "decorations": false,                  // 무테
        "shadow": false,                       // 그림자 제거
        "alwaysOnTop": true,                   // 항상 위
        "visibleOnAllWorkspaces": true,        // Spaces 전역
        "skipTaskbar": true,
        "contentProtected": true,              // 화면공유 비노출(※ macOS 15+ 무효)
        "focus": false,                        // 시작 시 포커스 안 뺏음
        "visible": false                       // 시작 시 숨김(단축키로 표시)
      }
    ]
  },
  "bundle": {
    "active": true,
    "targets": ["app", "dmg"],
    "macOS": { "minimumSystemVersion": "13.0" }
  }
}
설정 메모

투명은 transparent:true만으로 부족하고 app.macOSPrivateApi:true + Cargo macos-private-api feature가 함께 필요하다(App Store 부적격). 앱 은폐의 app.macOS.activationPolicy:"accessory"는 선언적 방법이며, 동일 효과를 런타임 set_activation_policy(ActivationPolicy::Accessory)로도 줄 수 있다(코드 골격 참조). contentProtected는 켜두되 macOS 15+에서는 효과가 없음을 전제로 한다. 위 예시의 // 주석은 설명용이며, 실제 tauri.conf.json(JSON)에는 주석이 허용되지 않으므로 제거한다.

④권한 (capabilities)

Tauri v2는 명령 호출 권한을 capabilities/*.json에서 창에 부여한다. 이 권한은 JS↔Rust IPC를 게이팅할 뿐, macOS OS 권한과는 무관하다(글로벌 단축키는 별도 OS 권한 불필요).

// src-tauri/capabilities/default.json
{
  "$schema": "../gen/schemas/desktop-schema.json",
  "identifier": "default",
  "description": "Peekaboo capability set",
  "windows": ["main"],
  "permissions": [
    "core:default",
    "core:window:allow-set-always-on-top",
    "core:window:allow-set-ignore-cursor-events",
    "core:window:allow-cursor-position",
    "core:window:allow-set-content-protected",
    "core:window:allow-set-focus",
    "core:window:allow-hide",
    "core:window:allow-show",
    "global-shortcut:allow-register",
    "global-shortcut:allow-unregister",
    "global-shortcut:allow-is-registered"
  ]
}

※ 첫 줄의 // src-tauri/... 주석은 파일 경로 표시용 — 실제 JSON 파일엔 넣지 않는다.

권한 식별자용도
core:window:allow-set-ignore-cursor-events클릭 통과 토글(Hover-Reveal)
core:window:allow-cursor-position커서 위치 폴링(핫존 히트테스트)
core:window:allow-set-content-protected화면공유 비노출(보조)
core:window:allow-set-always-on-top항상 위 토글
core:window:allow-set-focusAccessory 온디맨드 포커스
core:window:allow-hide · allow-show패닉 숨김 / 복귀
global-shortcut:allow-register 외패닉 단축키 등록/해제(기본 비활성 → 명시 필요)

⑤핵심 코드 골격

스텔스 본체(Rust)와 표현(프론트)의 최소 골격. 실제 구현 시 에러 처리·설정 주입을 더한다.

Rust — 플러그인 · 패닉 단축키 · 앱 은폐 (src-tauri/src/lib.rs)

use tauri::{Manager, ActivationPolicy};
use tauri_plugin_global_shortcut::{Code, Modifiers, Shortcut, ShortcutState, GlobalShortcutExt};

pub fn run() {
    tauri::Builder::default()
        .plugin(
            tauri_plugin_global_shortcut::Builder::new()
                .with_handler(|app, _shortcut, event| {
                    if event.state() == ShortcutState::Pressed {
                        if let Some(win) = app.get_webview_window("main") {
                            // 패닉 토글: Rust 핸들러에서 직접 → 저지연
                            if win.is_visible().unwrap_or(false) {
                                let _ = win.hide();
                            } else {
                                let _ = win.show();
                                let _ = win.set_focus();
                            }
                        }
                    }
                })
                .build(),
        )
        .setup(|app| {
            // Dock/Cmd-Tab 숨김 (선언적 설정 대신/병행 런타임 적용)
            #[cfg(target_os = "macos")]
            app.set_activation_policy(ActivationPolicy::Accessory);

            // 패닉 단축키 — 일반 조합(미디어키 금지 → macOS 권한 불필요)
            let panic_key = Shortcut::new(Some(Modifiers::SUPER | Modifiers::SHIFT), Code::KeyH);
            app.global_shortcut().register(panic_key)?;

            // 커서 폴링 루프 시작
            spawn_hover_loop(app.get_webview_window("main").unwrap());
            Ok(())
        })
        .run(tauri::generate_context!())
        .expect("error while running Peekaboo");
}

Rust — Hover 커서 폴링 루프

use std::{thread, time::Duration};
use tauri::{Emitter, WebviewWindow};

fn spawn_hover_loop(win: WebviewWindow) {
    thread::spawn(move || {
        let mut inside_prev = false;
        loop {
            if let Ok(pos) = win.cursor_position() {            // 데스크톱 절대좌표(클릭통과 무관)
                let inside = hotzone_contains(&win, pos);       // outer_position()+inner_size()로 rect 계산
                if inside != inside_prev {                      // 엣지에서만 토글
                    let _ = win.set_ignore_cursor_events(!inside);
                    let _ = win.emit(
                        "reveal-state-changed",
                        if inside { "revealed" } else { "ghost" },
                    );
                    inside_prev = inside;
                }
            }
            thread::sleep(Duration::from_millis(40));           // ~25fps 폴링
        }
    });
}

Frontend — 상태 수신 · 표현 동기화 (src/main.ts + styles.css)

import { listen } from "@tauri-apps/api/event";

const viewer = document.getElementById("viewer")!;
// Rust 상태머신이 보낸 전이를 받아 표현만 바꾼다
listen<string>("reveal-state-changed", (e) => {
  viewer.dataset.state = e.payload;   // "ghost" | "revealed" | "hidden"
});
/* styles.css — 투명창 위 콘텐츠 불투명도로 노출 제어 */
#viewer { transition: opacity .12s ease; }
#viewer[data-state="ghost"]    { opacity: .08; }  /* 평소 — 거의 투명 */
#viewer[data-state="revealed"] { opacity: 1;   }  /* 호버 — 또렷 */
#viewer[data-state="hidden"]   { opacity: 0;   }
구현 함정

① 글로벌 단축키 플러그인을 이중 초기화(자동 빌더 라인 + 수동 setup)하면 Invalid argument (os error 22) 패닉 — 1회만 등록. ② 단축키는 미디어키 금지(CGEventTap → 손쉬운사용 권한 유발). ③ cursor_position()은 desktop 전용이며, emit은 tauri::Emitter 트레이트 import가 필요. ④ Accessory에서 입력 받으려면 set_focus() 필수.

⑥빌드 · 서명 · 배포

개인용(미서명)과 배포용(서명·공증)을 구분한다. Peekaboo는 본질적으로 개인 도구이므로 미서명 경로가 현실적이다.

# 사전 준비
rustup default stable
xcode-select --install

# 스캐폴딩 & 플러그인
npm create tauri-app@latest peekaboo
cargo tauri add global-shortcut      # Rust + JS + 권한 항목 추가

# 개발 / 빌드
npm run tauri dev
npm run tauri build                  # → src-tauri/target/release/bundle (.app / .dmg)

# 개인용 미서명 실행 — Gatekeeper 격리 속성 제거
xattr -dr com.apple.quarantine "Peekaboo.app"

개인용 (권장 경로)

  • 미서명 또는 ad-hoc 서명으로 충분 — 본인 머신에서 실행.
  • Gatekeeper 경고는 xattr -dr com.apple.quarantine 또는 우클릭→열기로 우회.
  • Apple Developer 계정·공증 불필요 → 비용·흔적 최소.

배포용 (선택)

  • Developer ID 서명: bundle.macOS.signingIdentity + 하드닝/entitlements.
  • 공증(notarization): Apple ID 또는 API 키 환경변수로 tauri build 시 처리.
  • 주의 macos-private-api 사용으로 App Store 배포는 불가.

※ 서명·공증의 구체 환경변수/필드는 Tauri 버전에 따라 다를 수 있어 배포 시 공식 문서로 재확인한다(확인필요). 개인용 미서명 경로는 영향 없음.
